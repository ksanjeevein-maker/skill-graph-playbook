# Contributing

Thanks for improving `skill-graph-playbook`.

This repository stays useful when additions make the graph easier to traverse, not just larger.

## What To Contribute

Good contributions:

- clearer entrypoints and navigation
- stronger examples for real domains
- notes that explain tradeoffs or boundaries
- install or usage guidance for specific agent environments
- reference material that helps an agent decide what to read next

Avoid contributions that only add bulk:

- giant link dumps
- duplicate notes with slightly different wording
- deep folder nesting without a real domain boundary
- reference files that restate the same idea without new value

## Preferred Change Size

Small and coherent beats large and vague.

Good batches:

- one README improvement with a clear audience benefit
- one new reference note with one clear purpose
- one example-graph expansion that adds a navigable path

## Writing Rules

- keep notes small enough to scan quickly
- use links inside prose to explain why the next note matters
- keep frontmatter simple and meaningful
- make uncertainty explicit when a graph is still incomplete
- point to the system of record when facts are operational or time-sensitive

## Extending The Example Graph

When adding to an example graph:

1. update the entrypoint or relevant MOC
2. add only the next few notes needed
3. explain why each linked note matters
4. stop before the example becomes a full product knowledge base

## Pull Requests

Before opening a pull request:

1. explain the specific navigation or clarity problem you solved
2. keep the commit message tied to one meaningful change
3. mention any new files that a reader should start with first
