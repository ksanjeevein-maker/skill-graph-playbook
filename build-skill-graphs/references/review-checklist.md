# Review Checklist

Use this checklist before calling a skill graph "done enough" for real use.

## Orientation

- can a new reader find the right starting note quickly
- does the entrypoint explain scope and current focus
- do MOCs group ideas instead of dumping links

## Traversal

- do links explain why the next note matters
- are important paths reachable in one or two hops
- is there a clear difference between index, MOC, and node behavior

## Note Quality

- does each note hold one complete method, tradeoff, or claim
- are duplicated ideas merged or clearly distinguished
- is uncertainty visible where the graph is incomplete

## Boundaries

- are live facts kept out of markdown when they belong in a typed store
- does each note point to the source of truth when needed
- has the graph avoided pretending to be a database

## Maintenance

- would a future editor know which MOC to update when adding a note
- are the largest notes still cheap to scan
- is the structure small enough to keep trustworthy
