# Maintenance Checklist

Use this checklist when a graph starts to feel stale, crowded, or hard to trust.

## After Adding New Notes

- update the entrypoint if the new note changes where a reader should start
- update the relevant MOC if the cluster gained a new important path
- remove or merge notes that now duplicate each other

## During Regular Cleanup

- trim oversized MOCs back to routing and orientation
- merge notes that always travel together
- split notes that are hiding two different jobs
- mark unresolved areas instead of implying false completeness

## Freshness Checks

- verify that notes pointing to changing facts still reference the right system of record
- remove stale examples that no longer match the workflow
- keep example graphs small enough to stay believable

## Drift Signals

If you notice any of these, the graph needs maintenance:

- people keep reading the wrong first note
- MOCs have become long link dumps
- one note contains methods, records, and decisions all at once
- editors are afraid to change the graph because its structure no longer makes sense
