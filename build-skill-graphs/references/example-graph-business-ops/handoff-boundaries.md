# Handoff Boundaries

Use this note when work keeps bouncing between teams and nobody is sure who should own the next step.

The right handoff is based on responsibility, authority, and source of truth, not who currently feels the most pressure. Read [[runbook-drift]] when the written escalation path no longer matches actual behavior, and use [[triage-severity]] when the handoff question is really a disguised urgency question.

## Key Rule

Handoffs should happen at responsibility boundaries, not frustration boundaries.

## Inputs

- current owner
- next required capability
- source of truth for the decision

## Outputs

- whether to keep, escalate, or transfer ownership
