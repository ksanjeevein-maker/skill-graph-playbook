# Maintenance Keeps MOCs Trustworthy

The best graph can quietly decay if its routing notes stop reflecting what the rest of the system has become. Entry points and MOCs carry implied authority, so they need explicit maintenance.

Follow [[mocs-orient-attention]] when the issue is local cluster design, and read [[typed-stores-hold-live-state]] when the deeper problem is not stale navigation but stale copied data.

## Key Rule

Every meaningful expansion of a graph should update the entrypoint or the relevant MOC in the same pass.

## Inputs

- a growing note set
- one or more routing notes

## Outputs

- more reliable navigation
- fewer stale hubs
- better trust in graph structure
