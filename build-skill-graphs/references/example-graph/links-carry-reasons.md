# Links Carry Reasons

A bare list of links proves that notes are connected. A sentence with a link explains how they are connected and when the reader should follow the edge.

Follow [[progressive-disclosure]] when the main problem is context cost, and follow [[mocs-orient-attention]] when a growing cluster needs routing structure instead of more link density.

## Key Rule

Every important link should answer one of these questions:

- why should the reader go there now?
- what changes after reading it?
- what risk, method, or tradeoff does it clarify?

## Inputs

- an atomic note with at least one adjacent concept

## Outputs

- more meaningful traversal
- less accidental over-reading
- clearer graph semantics
