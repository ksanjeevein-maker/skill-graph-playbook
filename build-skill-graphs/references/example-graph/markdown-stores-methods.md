# Markdown Stores Methods

Markdown is strongest when it holds the part of the system that needs explanation, context, and traversable reasoning. It works well for reusable methods because links can show how a pattern relates to adjacent ideas.

Read [[typed-stores-hold-live-state]] when the content starts behaving more like an operational record than a reusable method, and use [[links-carry-reasons]] when the note needs clearer routing to nearby concepts.

## Key Rule

If the main value of a note is how it helps the agent think, compare, or navigate, markdown is usually the right home.

## Inputs

- stable guidance
- methods and heuristics
- synthesized claims

## Outputs

- reusable knowledge
- richer traversal
- lower coupling to live system changes
